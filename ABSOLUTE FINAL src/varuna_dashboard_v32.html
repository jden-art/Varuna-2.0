The National Infrastructure Gap

India's Flood Warning Gap Is a Solvable Engineering Problem

Of India's 412 flood-prone districts, fewer than 30% have functioning automated water-level monitoring. The rest rely on manual gauging or nothing at all. Project Saraswati is engineered to close this gap in a single phased programme — deploying from a district pilot to a 5,000-node national grid within 18 months.

412
Districts at High Risk
NDMA-identified flood-vulnerable districts. Each requires a minimum of 8–12 monitoring nodes for adequate riverine coverage across all tributaries and embankments.
288
Districts with Zero Automation
288 at-risk districts have no automated early-warning infrastructure whatsoever. Field officers rely on visual inspection and telephone reports to gauge flood risk.
82%
Deaths in Unmonitored Areas
The overwhelming majority of flood fatalities occur in districts with no automated monitoring. Warning time is the single most important factor in survival outcomes.
5,000
Target Network Size
Full national deployment of 5,000 VARUNA nodes across all 412 districts — achievable in Phase 3, 12–18 months from programme start. Each node costs a fraction of a legacy gauging station.
Network Architecture

One Architecture. Scales from 50 to 5,000 Nodes.

Each VARUNA sensor node is a fully autonomous monitoring station. Thousands of nodes push telemetry through Firebase cloud into a unified hierarchy of district, state, and national dashboards — with no single point of failure at any layer.

VARUNA Node Capabilities

What Every Node in the Network Must Do

Each VARUNA node is a self-contained, solar-powered monitoring station. It senses, classifies, alerts, and reports — continuously and autonomously, without requiring internet connectivity for safety-critical flood warnings.

01

Real-Time Water Level

HC-SR04 ultrasonic sensor. Continuous 1 Hz sampling. Basin-specific alert thresholds. Seasonal calibration offsets supported.

HC-SR04 · 2–400 cm · ±1 cm · 1 Hz
02

On-Device Flood Classification

4-tier algorithm runs entirely on the ESP32-S3. Triggers SMS and voice alerts even when internet is unavailable. No cloud dependency for warnings.

4-tier: Normal/Alert/Warning/Danger · Offline-capable · 45 min+ lead
03

Multi-Channel Community Alerting

SIM800L GSM sends SMS and voice alerts simultaneously to village responders, BDOs, tehsildars, and SDMA control rooms upon any zone escalation.

SIM800L · SMS + Voice · SDMA + Revenue Dept routing
04

GPS-Georeferenced Telemetry

Every packet carries GPS coordinates and UTC timestamp. Enables GIS integration, NDRF deployment mapping, and post-flood basin analysis.

NEO GPS · <2.5m CEP · 10 Hz · NMEA 0183
05

38-Parameter Telemetry Stream

38-field CSV per second: water height, tilt X/Y, temperature, pressure, battery, WiFi RSSI, GPS, satellite count, flood zone, response level, and 25 more fields.

38 fields · 1 Hz · Firebase RTDB · CSV/Excel export
06

Structural Integrity (IMU)

MPU6050 6-axis IMU detects embankment tilt, bridge stress, and displacement events — signalling infrastructure risk before water levels become critical.

MPU6050 · 6-axis · ±250°/s · ±2g · Tilt + displacement alerts
07

Remote OTA Firmware

All nodes receive firmware updates from the central dashboard. Critical for a 5,000-node network — no physical access needed for software maintenance.

Web Serial API · .bin push · All nodes simultaneously
08

Dual-Link Connectivity

WiFi primary with GSM/GPRS fallback via SIM800L. Data continuity even when local infrastructure is flood-damaged. Node alerts via SMS in full offline mode.

WiFi 2.4GHz + GSM/GPRS fallback · SMS offline mode
Field Deployment Playbook

From Factory to Field in Under 72 Hours

Each VARUNA node ships pre-configured with Firebase credentials, district flood thresholds, and alert contact lists. A trained field technician installs and commissions a node in under 2 hours, with no specialist equipment.

01

Site Survey & Node Registration

District technical officer identifies installation point — bridge parapet, embankment, sluice gate. Node registered in national inventory with GPS coordinates, basin classification, and flood history data.

Responsible: District Engineer / SDMA Technical Cell
02

Physical Installation

Node mounted on galvanised pole or bridge parapet using standard hardware. Ultrasonic sensor aimed at water surface. Solar panel oriented. BSNL/Jio SIM inserted with registered data plan.

Time: <2 hours · Tools: Standard installation kit
03

Commissioning & Calibration

Field officer accesses commissioning checklist on dashboard via mobile browser. Node verified live. Baseline water-level offset calibrated. Zone thresholds set against local 10-year flood records. First telemetry confirmed.

Verification: Dashboard commissioning checklist · <30 min
04

Alert Routing & Integration

SMS alert contacts registered: BDO, tehsildar, village panchayat, SDMA control room, NDRF liaison. Node integrated into district GIS dashboard and NDMA reporting pipeline. Integration test completed.

Integrations: SDMA · NDMA · Revenue Dept · NDRF
Remote Network Health
Dashboard automatically flags nodes that go offline, report low battery, or return anomalous sensor data. Field maintenance is dispatched only when the system determines it is needed.
Centralised OTA Firmware
Security patches and algorithm updates pushed to all 5,000 nodes from a single dashboard OTA session. No field visit required. Rollback capability built in.
Automated Archiving & Reporting
All telemetry archived in Firebase. One-click CSV/Excel export for post-flood event analysis, budget audit, and mandatory CWC/NDMA compliance reporting.
Government Interoperability

Plugs Into India's Existing Disaster Management Ecosystem

Project Saraswati does not replace existing infrastructure — it fills the critical gaps. Every data format, alert protocol, and access tier is designed to integrate with the agencies already responsible for flood management.

NDMA — National Disaster Management Authority
Real-time alert feeds compatible with National Disaster Risk Reduction Portal (NDRRP) and India Disaster Resource Network (IDRN). Role-based access for NDMA operations centre.
CWC — Central Water Commission, Hydrology Division
Telemetry format aligned with CWC FloodWatch data standards. Direct CSV export compatible with Water Resources Information System (WRIS) ingest pipeline.
IMD — India Meteorological Department, Flood Met Division
BMP280 atmospheric pressure and temperature data formatted for compatibility with IMD's surface observation network. Augments IMD rainfall correlation analysis.
SDMAs — All State Disaster Management Authorities
Per-state dashboard tier with role-based access for SDMA control rooms, district collectors, and block development officers. Alert routing customisable per district.
NDRF — National Disaster Response Force
Critical zone mapping from live node network data feeds NDRF pre-positioning intelligence. Danger-zone clustering enables battalion deployment before flood peaks arrive.
CAP v1.2 — Common Alerting Protocol
Alert output compatible with CAP v1.2 standard, enabling integration with Doordarshan Emergency Warning System, All India Radio, and Cell Broadcast EWS feeds.
